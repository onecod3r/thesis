# 2026-07-18 — Repo restructure: unified training stack, flat epoch-seconds registry, data-tree policy

Full-repo restructure (TODO §0.4). No training or analysis today — everything
below is tooling/layout; all changes smoke-tested (imports, forwards +
causality probes for all 4 architectures, registry round-trip, notebook JSON +
syntax, index rebuild, eval-script bootstrap).

## 1. Unified training stack — `src/modules/model/`

The four `gislr.1.model.*.ipynb` notebooks were ~95% identical. All shared
logic now lives in one package:

| module | content |
|---|---|
| `architectures.py` | `StreamingGRU` / `StreamingLSTM` / `BiLSTM` / `CausalConv1D` + the `ARCHS` registry — the **single** definition, imported by both training and eval (state_dicts can no longer drift) |
| `data.py` | canonical split (asserted 9,448 val), per-subset feature caches → `data/cache/gislr/features/`, in-RAM dataset |
| `registry.py` | run folders, `meta.json` writing (schema check + canonical-field preservation), asset registration, pointer files → `data/cache/runs/` |
| `train.py` | the driver: auto-resume, early stopping, ReduceLROnPlateau, **one tqdm bar per run** (batch progress, losses, LR, plateau counter all in the same bar — no nested bars, no per-epoch print spam), `meta.json` rewritten every epoch |
| `report.py` | learning curves into a run's `assets/` |

The notebooks are now thin drivers: architecture-identity block (`ARCH`,
`TRAIN_SUBSETS`, `HYP`) + calls into `modules.model`. Regenerated from one
template, so they cannot drift from each other again.

## 2. Model registry v2 — flat, epoch-seconds, meta.json-only

- New layout: `src/data/models/<run_id>/` with **run_id = seconds since the
  Unix epoch** at training start. Flat — dataset/architecture/subset are
  `meta.json` fields and `index.csv` columns, not directory levels.
- Run folder contents: `meta.json` + `best.pt` + `last.pt` (gitignored) +
  `assets/` (everything else, each file linked from `meta.json["assets"]`).
  Per-run `README.md`/`data.md`/`cache/` are gone — the structured record
  replaces the narrative one.
- **Schema v2** documented in README.md § "meta.json schema" (source of
  truth); machine check in `registry.py::REQUIRED_KEYS`. Canonical-eval fields
  written by `eval_gru.py` survive training-loop rewrites.
- **Registry reset**: started empty. The 8 pre-reset runs survive only in git
  history (≤ `3668dae`, old `src/models/` tree) and the daily reports. Their
  weights were deleted with the old tree, so the 6 pending canonical evals
  can never be completed — the v1 numbers (FULL_543 70.59, ME_126 73.73 xyz /
  74.92 xy train-loop, …) stand as historical references only.

## 3. Data tree + placement policy

`src/data/{raw, cache, temp, external, models}` with `modules/paths.py` as the
single source of tree constants — now **absolute/CWD-independent** and with
**lazy** dataset resolution (`gislr_dir()`, `resolve_datasets()`; importing
`modules.paths` no longer downloads anything).

Policy: extracted-from-source → `data/raw/<dataset>/`; reusable derived →
`data/cache/<dataset>/`; throwaway → `data/temp/` **deleted after use**
(`cleanup_temp()`); third-party assets → `data/external/`. `.gitignore` ignores
all of `src/data/` except the registry (`data/models/`, minus `*.pt` and
`export/`).

Applied: POPSIGN pilot npz now goes to `data/temp/popsign_pilot/` with a
cleanup cell in `popsign.0` (the stale 10 MB `data/raw/popsign/_pilot` tree
was deleted); extraction manifests move to `data/cache/popsign/dataframes/`
(**need regeneration** — the old copies were cleared in the restructure);
`gislr.0.*` diagnostic caches move to `data/cache/gislr/{motion_analysis,
subset_comparison}` (also cleared → re-runs recompute).

## 4. Scripts split

`src/modules/scripts/` = project Python CLIs (`eval_gru.py`, rewritten for the
new layout: takes a run folder, imports models from `modules.model`, writes
into `assets/` + `meta.json`; `build_model_index.py`, rewritten for the flat
registry, header-only CSV on empty). Both bootstrap `sys.path` and run from
any CWD. Root `scripts/` keeps housekeeping only (`sys_disk_usage.ps1`).

## 5. Docs split

`docs/daily/<YYYY-MM-DD>.md` (day logs, this file), `docs/weekly/<YYYY>-<WW>.md`
(ISO week, e.g. `2026-29.md`), `docs/reports/<topic>.md` (standalone topic
reports) — convention recorded in `docs/README.md`.
